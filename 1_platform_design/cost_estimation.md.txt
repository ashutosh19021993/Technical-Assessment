Cost Estimation (AWS eu-central-1)
Overview

This section provides an estimated monthly cost for running the multi-tenant platform in the AWS eu-central-1 (Frankfurt) region. The estimate is based on a baseline shared platform supporting approximately 20+ teams, with the ability to scale.

The goal is to:

provide a realistic cost baseline

identify major cost drivers

highlight optimization strategies

Assumptions

The estimation is based on the following setup:

1 Amazon EKS cluster (standard support)

3 worker nodes (medium-sized instances)

1 NAT Gateway

1 Application Load Balancer (ALB)

1 small RDS PostgreSQL instance

moderate S3 usage

OpenTelemetry + New Relic integration

workloads distributed across 3 AZs

Cost Drivers

The main contributors to cost are:

worker nodes (compute usage)

RDS instance size and HA configuration

NAT Gateway data processing

observability data ingestion

number of workloads and traffic volume

Cost Scaling (20 → 50+ Teams)

As the platform grows:

more nodes will be required → increased EC2 cost

higher traffic → increased ALB and NAT cost

more telemetry → higher observability cost

some teams may require dedicated infrastructure → increased cost per tenant

Estimated scaled cost:

~$1000–3000+/month, depending on workload size and isolation requirements
